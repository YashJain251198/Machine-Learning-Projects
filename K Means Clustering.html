K Means Clustering Vs Hierarchical Clustering¶

  1. Visual Stand Point - We have Elbow Plot to let us know how many clusters in K Means where as in Hierarchical Clustering, we have Dendrograms that tell us how many clusters can be formed from the data itself.
  1. In K Means Clustering, we need to have K Value in Hand for creation of Clusters which means K Value drives the clusters however, in Hierarchical Clustering, there is no Need of K Value as the Hierarchical Clustering works on the theme of Hierarchy.
  1. The linkage between the data points in K Means Clustering is done by EUCLIDEAN DISTT however we have several Linkage Approaches in H. Clustering.
  1. The K Means is divided in Soft and Hard Clustering however, the H. Clustering is divided in Agglomerative Clustering and Divisive Clustering.

Role of Elbow¶

Since the K Value by default is 8 which means that it may not be the correct the cluster count and hence we would need the ELBOW Method Approach to find the Correct Cluster Count.

Steps

  1. We will create ELBOW Plot to find the Optimal Cluster Count.

  2. We will take the Cluster Centres and plot in the Data to see the Centroids in the Plot
